FAQ

How did you pick which sites to test with?

We tried to use the most popular sites in each category according to their Alexa rating. If there's a filter in place, it should at least spot these sites.

We require sites that have a favicon since browsers allow us to fetch them from arbitrary domains and check that we received a valid image. Filters may return 404s, 403s, DNS errors, or return a HTML 'this is blocked' page. By fetching a favicon we can detect all these possibilities.

Where possible, we use HTTPS addresses. This allows us to eliminate filters that forbid 'http://bad.example.com/' but allow 'http://bad.example.com/favicon.ico', since it is not possible for routers to see the path over HTTPS connections.

Can users evade a filter?

Yes. There are many approaches, so we only cover the two most common approaches here.

There are many websites (e.g. [1] [2]) available that enable you to circumvent filtering. Of course, some filters also block web proxies. There is a cat-and-mouse game between filter tools and online proxies, and the filter tools frequently lose. Users only need to find one accessible proxy, whereas a filter tool must block all of them, with new proxies appearing often.

You can also connect to the Internet through a VPN provider (e.g. [1] [2]). Using a VPN, your local ISP cannot see what websites you are accessing, so cannot filter.
